# 聊天段子短视频自动化流水线 · 操作手册

一条命令把「模拟微信聊天段子」变成 1080×1920 竖屏成片：
LLM 写段子 → 逐条魔性配音 → 表情包/音效/BGM → 逐帧渲染 → ffmpeg 合成。

---

## 一、环境要求

| 依赖 | 说明 |
|---|---|
| Python 3.10+ | 已验证 3.12（conda 环境 `py12-sweet`） |
| ffmpeg / ffprobe | `brew install ffmpeg`，已验证 8.1 |
| Python 包 | `pip install -r requirements.txt`（edge-tts、playwright、requests） |
| Chromium 内核 | `playwright install chromium`（仅首次） |
| OpenRouter key | 写在项目根 `openrouter.md` 里（`sk-or-` 开头），或环境变量 `OPENROUTER_API_KEY` |

注意：edge-tts 和 OpenRouter 都需要联网；渲染与合成完全本地。

## 二、目录结构

```
short-video/
├── pipeline/            # Python 管线包（核心代码）
│   ├── config.py        #   全局配置：音色、节奏参数、默认模型 ← 调参看这里
│   ├── gen.py           #   入口：LLM 批量生成段子
│   ├── make.py          #   入口：段子 JSON → MP4
│   ├── script_gen.py    #   段子生成提示词（爆款公式）← 调"段子风格"看这里
│   ├── tts.py / timeline.py / render.py / compose.py / assets.py / llm.py / media.py
├── renderer/renderer.html  # 确定性渲染器（虚拟时钟 seek，逐帧截图用）
├── assets/              # 素材库（已配齐，可持续补充）
│   ├── sfx/             #   30+ 个音效 + manifest.json（基础 + 魔性笑声 + 怪音）
│   │   ├── builtin/     #   代码自动合成的兜底音效（勿删）
│   │   └── voicelines/  #   中文梗语音（你干嘛/雪豹/魔性笑），tag 带 vl_ 前缀
│   ├── bgm/             #   14 首 + manifest.json：轻快沙雕 6 + 唢呐(suona) + 悲怆钢琴(sad) + 紧张渐强(tension)
│   ├── images/          #   AI 生成的角色头像与剧情道具图
│   │                    #   头像库：avatar_gf/bf(正常) avatar_crazy_m/f(疯癫猥琐风)
│   └── memes/<情绪>/    #   76+ 张 13 类情绪梗图
├── scripts/             # 段子 JSON（手写的和 LLM 生成的都在这）
├── output/<片名>/       # 成片 <片名>.mp4 + 封面 cover.jpg + 配音 audio/
└── chat-simulator.html  # 最早的手动版（保留，可浏览器里手动玩）
```

## 三、日常使用

### 1. 全自动：给个话题直接出片

```bash
cd /Users/jason/works/ai/short-video
python3 -m pipeline.gen "相亲对象的奇葩操作" -n 10
```

每批生成进**独立子目录** `scripts/<月日_时分>_<主题>/`，自动附带毒舌评分：
- 文件按分数排序命名：`01_33分_标题.json`（40 分制，<30 标"毙"）
- `索引.md`：每个本子的钩子、毒评、最该改的一条、出片命令——**挑片只看这个文件**
- 加 `--make` 直接出片（只出 ≥30 分的）；加 `--no-judge` 跳过评分省一次调用

建议日常用法：先生成一批看索引，**人工挑 1~2 个手动改两句**再出片——评委的"建议改"经常值得采纳。

### 2. 半自动：手写/改段子再出片

```bash
python3 -m pipeline.make scripts/我的段子.json
python3 -m pipeline.make scripts/*.json            # 批量
python3 -m pipeline.make scripts/x.json --keep-frames   # 保留帧序列便于调试
```

约 30 秒的片子全流程耗时 ≈ 2 分钟。

### 3. 段子 JSON 怎么写

参考 `scripts/demo_诈骗电话.json`，完整字段：

```jsonc
{
  "title": "内部文件名用",
  "hook": "视频顶部大字标题，≤14字，要有好奇缺口",
  "intro": { "text": "注意看式开场白≤18字" },     // 前3秒：黑屏大字+魔性解说音，决定完播率
  "cover": {                                       // 封面：土味大字报，自动渲染 cover.jpg
    "title": "封面大字（\\n 可换行）",
    "sub": "红条副标题",
    "image": "情绪tag或图片路径"                   // 缺省自动用 right 头像
  },
  "chat_title": "聊天窗口标题",
  "left":  { "avatar": "assets/images/avatar_gf.png", "bg": "#ffd6e7" },  // 对方；avatar 支持 emoji 或图片路径（图片更拟真，可 AI 生成）
  "right": { "avatar": "assets/images/avatar_bf.png", "bg": "#c9e4ff" },  // 自己（绿气泡一侧）
  "time_label": "晚上 11:24",                     // 聊天顶部灰色时间戳，可省略
  "bgm": "silly",        // silly/sneaky/scheming/goofy/chaos/upbeat，省略=自动选
  "endcard": "关注我，下集更炸 🔥",               // 结尾贴片，可省略
  "messages": [
    { "side": "left",  "text": "普通消息" },
    { "side": "right", "text": "憋大招的消息", "typing": true },   // 出现前显示"正在输入"
    { "side": "left",  "meme": "shock" },                          // 插表情包（按情绪随机抽）
    { "side": "left",  "text": "反转句", "punch": true, "sfx": "dun" },  // 高潮：缩放震动+重音
    { "side": "right", "text": "哈哈哈哈哈", "sfx": "laugh" },     // TTS 真念哈哈哈，自带笑点
    { "side": "left",  "text": "补刀", "pause": 600 },             // 额外停顿 ms
    { "cutaway": "cry", "caption": "家人们，这是什么纯爱战神", "sfx": "vl_haha" }
    // ↑ 全屏插播：黑屏+大图+大字旁白打断聊天，caption 由魔性解说音念出
    //   cutaway 填 meme 情绪或图片路径；每片 1~2 次，用在反转炸开/离谱发言之后
  ]
}
```

- `sfx` 基础：`laugh` 罐头笑 / `boom` / `suona` 唢呐 / `dun` 重音 / `whoosh` / `ding` / `fail` / `applause` / `awkward` 蛐蛐 / `scratch` 急刹
- `sfx` 魔性扩展：`laugh_evil` 奸笑 / `laugh_devil` 低沉魔笑 / `laugh_duck` 鸭笑 / `laugh_baby` 婴儿笑 / `laugh_squid` 海绵宝宝笑 / `goose` 鹅叫 / `bruh` / `damage` 暴击 / `error` 错误音 / `wow` / `huh` / `boing` 弹簧 / `pop_cat` / `drumroll` 揭晓鼓点 / `tada` / `slap` 拍打
- `sfx` 中文梗语音：`vl_nigan` 你干嘛 / `vl_wudi` 雪豹 / `vl_haha` 魔性笑
- `bgm` 可用：`silly / sneaky / scheming / goofy / chaos / upbeat`（轻快）+ `suona` 百鸟朝凤 / `sad` 悲怆钢琴(反差用) / `tension` 山魔王宫殿(渐强压迫)
- `meme` 情绪：`shock / speechless / laugh / cry / doge / angry / smug / confused / money 要钱 / scheme 算计 / no 拒绝 / love 舔狗 / tired 摆烂`（随机抽取；关键剧情位建议用 `image` 直接钉死某张图，避免抽到不合适的）
- 消息级还可覆盖 `voice` / `rate` / `pitch`（见下）
- 脚本级节奏字段（控制总时长）：`bgm_tempo`（BGM 倍速，1.2 更魔性）、`pad_after_audio`（默认 380ms，紧凑可设 220）、`image_dwell`（表情包停留，默认 1600）、`end_hold`（结尾定格，默认 2400）
- 参考 `scripts/彩礼反转_新娘是小芳.json`：约 31 秒的紧凑节奏配置范例

## 四、自定义与调参

### 换配音音色（魔性之源）

编辑 `pipeline/config.py` 的 `VOICES`，或在角色/消息上写 `"voice"`：

```jsonc
"left": { "avatar": "🐱", "bg": "#ffb347", "voice": "dongbei" }          // 整个角色东北话
{ "side": "left", "text": "干哈呢", "voice": "dongbei", "rate": "+25%" } // 单条消息覆盖
```

可用别名：`dongbei`（东北话·搞笑）、`shaanxi`（陕西话）、`narrator`（加速魔性解说）。
也可直接写 edge-tts 音色 id（`python3 -m edge_tts --list-voices | grep zh` 查看全部）。
**rate 加快 + pitch 升高 = 更魔性**，如 `"rate": "+35%", "pitch": "+20Hz"`。

### 调节奏 / 特效

`pipeline/config.py`：`PAD_AFTER_AUDIO`（配音后呼吸）、`IMAGE_DWELL`（表情包停留）、`PUNCH_EXTRA`、`END_HOLD`。
punch 的缩放幅度/震动在 `renderer/renderer.html` 的 `seek()` 里（`0.13` 即放大 13%）。
BGM 铺底音量：`pipeline/compose.py` 的 `build_master_track(bgm_volume=0.16)`，想要 BGM 更突出可调到 0.2~0.25（配音响起时会自动闪避，不用担心压过人声）。

### 调段子风格：script_gen.py 完全指南

`script_gen.py` 是段子的「创作引擎」，**不直接运行**——入口永远是 `python3 -m pipeline.gen "话题"`，它在背后做三件事：`generate()` 调 LLM 写稿 → `_validate()` 校验修正 → `judge()` 毒舌评分 → `save_batch()` 存批次目录。

日常你只需要编辑文件顶部的**三段提示词**（都是普通字符串，改完即生效、无需重启）：

| 变量 | 作用 | 什么时候改 |
|---|---|---|
| `SYSTEM` | 创作规则：笑点武器库、反转铁律、硬性禁令 | 想整体换风格（如从"癫狂"转"温情反转"）；发现模型反复犯某种毛病就加一条禁令 |
| `EXAMPLE` | 爆款结构拆解（few-shot 锚点）+ 反面教材 | **最常改、效果最大**。每出一条爆款就回流进来 |
| `JUDGE_SYSTEM` | 评委的打分口味与毙稿线 | 觉得评分太松/太严；想加评分维度（如"梗的新鲜度"） |

**爆款回流的具体操作**（提质最有效的一招）：发布后某条爆了，打开 `script_gen.py` 找到 `EXAMPLE`，仿照已有条目追加一条结构拆解，格式如下：

```
④ 云驱鬼：怕鬼→"开灯，放国歌"→9.9在线驱鬼→视频里是他自己穿道袍(punch)
   →"五星好评返2块"（神回复开场 + 反向创业流 + 返现补刀）
```

要点：写**结构和笑点机制**而不是全文照抄；末尾括号里标注用了武器库里的哪几招。锚点保持 3~6 个，太多会稀释。

**其他可调项**：
- `SFX_TAGS`：新增音效素材后把 tag 加进这个列表，模型才知道能用
- `_validate()` 里的消息数上下限（6~18）、单条字数上限（36）
- 评分毙稿线：`gen.py` 里 `--make` 只出 ≥30 分的，阈值在 `gen.py` 末尾改

**进阶：在自己的脚本里调用**

```python
from pipeline import script_gen
scripts = script_gen.generate("话题", n=5)          # 返回已校验的脚本 dict 列表
scores  = script_gen.judge(scripts)                  # 毒舌评分 [{index,total,comment,fix,verdict}]
batch_dir, paths = script_gen.save_batch("话题", scripts, scores)  # 存批次目录，返回 [(路径,分数)]
```

### 换模型

```bash
python3 -m pipeline.gen "话题" -n 5 --model anthropic/claude-sonnet-4.6
```

默认 `deepseek/deepseek-v4-pro`（¥0.03/批）。**实测对比结论（2026-06-11，同题同提示词）**：
- deepseek：能蹦出"宇宙民政局""萝卜章公证件"这种高光癫句，但反转工程粗糙、常超长被毙，成品率低（1/4）
- Sonnet 4.6（¥0.25~0.35/批）：信息差反转埋得干净（"我是快递站的，你的手机在这里签收一下"）、金句收尾稳、成品率 4/4
- **建议：正经出片用 `--model anthropic/claude-sonnet-4.6`，头脑风暴/堆量用默认 deepseek**。评分环节固定用 deepseek（判别任务，便宜够用）。

### AI 生成剧情道具图（如"婚礼现场照"）

剧情需要的特定图片（聊天里发的"照片"）可用 OpenRouter 图像模型生成，存进 `assets/images/` 长期复用：

```python
# python3 运行；模型 google/gemini-2.5-flash-image 便宜好用（约 $0.0003/张）
import base64, requests, re
key = re.search(r'sk-or-[A-Za-z0-9-]+', open('openrouter.md').read()).group(0)
r = requests.post("https://openrouter.ai/api/v1/chat/completions",
    headers={"Authorization": f"Bearer {key}"},
    json={"model": "google/gemini-2.5-flash-image",
          "messages": [{"role": "user", "content": "你的图片描述（注明：除XX外不要出现任何文字，避免乱码）"}],
          "modalities": ["image", "text"]}, timeout=120)
b64 = r.json()["choices"][0]["message"]["images"][0]["image_url"]["url"].split(",", 1)[1]
open("assets/images/道具名.png", "wb").write(base64.b64decode(b64))
```

然后在消息里 `{"side": "right", "image": "assets/images/道具名.png", "punch": true}`。

### 补充素材

- 音效：丢 mp3 进 `assets/sfx/`，在 `assets/sfx/manifest.json` 加 `{"file","tag","source"}`；文件名包含 tag 也能被匹配
- BGM：同理放 `assets/bgm/`
- 梗图：直接丢进 `assets/memes/<情绪>/`，无需登记，自动随机抽取
- 全部素材永久保留、跨视频复用；manifest 里 file 写相对所在目录的文件名即可

## 五、产出物

每条片在 `output/<片名>/`：

- `<片名>.mp4` — 1080×1920 / 30fps / h264+aac / 响度 -14 LUFS（抖音、视频号、B 站竖屏可直发）
- `cover.jpg` — 渲染器排版的大字报封面（放射线底+黄字标题+主图+红条副标题）；脚本没写 cover 字段时退化为反转帧截图
- `audio/` — 逐条配音 mp3（复用/检查用）

## 六、常见问题

| 现象 | 处理 |
|---|---|
| TTS 报错/超时 | 网络问题；内置 3 次重试，仍失败换网络再跑（edge-tts 需访问微软服务） |
| 生成段子解析失败 | 模型偶发输出畸形 JSON，重跑一次；或换 `--model` |
| 表情包没出现 | 对应情绪目录为空 → 该条消息自动跳过（不会出空气泡）；补图即可 |
| BGM 没按 tag 选中 | 检查 `assets/bgm/manifest.json` 的 tag 拼写；找不到时自动选目录里第一首 |
| 想看中间帧调试 | `--keep-frames`，帧在 `output/<片名>/frames/` |
| 渲染慢 | 正常 ~2 分钟/条；可降 `config.FPS` 到 24 提速 |
| 改完代码想回归验证 | `python3 -m pipeline.make tests/stress_回归.json`（覆盖纯标点消息、typing、表情包、punch 等全特性） |

## 七、批量生产建议（高质量自动化的正确姿势）

1. **早上**：`python3 -m pipeline.gen "<热点话题>" -n 10`（话题蹭当天热搜）
2. **人工 5 分钟**：翻 `scripts/` 里的 JSON，挑 2~3 个真好笑的，顺手改两句台词
3. **出片**：`python3 -m pipeline.make scripts/挑中的.json`
4. **发布后**：把爆了的段子加进 `script_gen.py` 的 EXAMPLE，模型会越写越像你的爆款

机器负责量产和工艺，人只守两道关：**挑段子、挑成片**。

## 七点五、魔性短片线（mode: magic）

第二条业务线：5~20 秒「抽象魔性」全屏短片——AI 魔性图片 + 暴力运镜 + 大字幕 + 魔性旁白 + 音效轰炸。与聊天线共用出片入口、配音、混音、封面、插播全套设施。

### 用法

```bash
python3 -m pipeline.gen "猫主子的早晨统治" --magic -n 4    # LLM 生成分镜（同样进批次目录+评分）
python3 -m pipeline.make scripts/xxx/01_xx分_xxx.json      # 出片（自动识别 mode）
```

参考范例：`scripts/demo_魔性健身猫.json`（7 镜把全部运镜各用一遍）。

### 分镜 JSON 关键字段

```jsonc
{
  "mode": "magic",                       // 必填，触发魔性管线
  "intro": {...}, "cover": {...},        // 与聊天线同款
  "bgm": "chaos", "bgm_tempo": 1.3,      // 短片线 BGM 底量默认 0.26（可用 bgm_volume 覆盖）
  "shots": [{
    "image": "gen:图片提示词",            // 图源：gen:AI生图 / refs/手动图 / meme:情绪
    // 动态底片三选一（构图统一为：虚化背景填充+主体居中）：
    //   "video": "refs/x.mp4"            本地视频直接当底片
    //   "image": "xxx.gif"               动图自动转码成动态底片
    //   "animate": true                  AI 图生视频（需配 DashScope key，见下）
    "motion_prompt": "画面如何动≤30字",   // animate 时的运动描述
    "motion": "punch_in",                // 运镜预设，见下表
    "caption": "大字≤8字",               // 屏幕大黄字
    "narration": "魔性旁白≤16字",         // narrator 解说音，时长驱动镜头时长
    "sfx": "boom", "sticker": "meme:doge", // 音效 + 表情包角标飞入
    "dur": 2000                          // 无旁白时的镜头时长 ms
  }]
}
```

### 图生视频（让画面真的动起来）

走 **OpenRouter 视频生成 API**，用项目现有 key，无需任何新账号。镜头加两个字段即可：

```jsonc
{ "image": "gen:...", "animate": true,
  "motion_prompt": "画面如何动≤30字（如：猫狂吼举杠铃过头，闪光灯爆闪）",
  "video_model": "google/veo-3.1-lite" }   // 可省略，默认即此
```

模型与单价（按输出秒计费，5 秒镜头参考价）：

| 模型 | 5秒成本 | 说明 |
|---|---|---|
| `google/veo-3.1-lite`（默认） | ≈$0.2 ≈¥1.4 | 已实测，质量好，720p 竖屏 |
| `alibaba/wan-2.6` | ≈$0.5 | 通义万相 |
| `kwaivgi/kling-v3.0-std` | ≈$0.42 | 可灵 |
| `x-ai/grok-imagine-video` | ≈$0.25(480p) | 最便宜，分辨率低 |

要点：
- 生成片段按（图+提示词+时长+模型）哈希缓存在 `assets/clips/`，同输入不重复扣费
- 每片只给反转/高潮镜头上 animate（1~2 个），其余镜头静图运镜，一条片成本可控在 ¥1.5~3
- 调用失败/欠费时该镜头自动退回静图运镜，整片不会报错
- GIF 动图、本地 mp4 当底片不花钱，优先用

### 运镜预设

| motion | 效果 | 用在 |
|---|---|---|
| `punch_in` | 急推放大 | 强调、开场 |
| `pull_back` | 反拉揭全景 | 反转揭晓 |
| `shake` | 衰减震动 | 冲击瞬间 |

### OpenRouter key 的用途与消费边界（已核实）

**本项目对 OpenRouter key 的全部用途只有两个**：deepseek 写段子、gemini 生成图片——均为管线代码从 `openrouter.md` 读 key 后直接 HTTP 调用。整个开发期项目在该 key 上的实际消费约 **$0.02**（1 次段子批生成 + 4 张图）。

**Claude Code 开发会话不使用这个 key**：开发走 Claude Max 订阅（订阅内不按 token 计费）。注意 `~/.zshrc` 里的 `claude-or()` 函数会把 Claude Code 切到 OpenRouter 按 Opus API 价（$5/$25 每百万）计费——**做本项目开发请用普通 `claude` 启动，不要用 `claude-or`**。如在 openrouter.ai/activity 看到 `anthropic/claude-opus-*` 的大额消费，即来自 `claude-or` 启动的会话，与本管线无关。

### 省钱守则

1. **日常量产只跑命令行**（`pipeline.gen` / `pipeline.make`），成本就是上表的几分钱
2. 开发/调优用普通 `claude` 启动（走 Max 订阅），不要用 `claude-or`
3. Claude 会话里的多代理工作流虽不直接花钱（订阅内），但消耗订阅额度较快，深度打磨时再开
4. OpenRouter 充值含约 5% 平台手续费，知情即可
5. 道具图、头像生成一次永久复用，放 `assets/images/` 别删
